# CNSVintraday V1.2 Feature Engine 开发文档

版本：

V1.2

阶段：

Feature Engine

前置条件：

V1.1.1 已验收通过。

目标：

建立盘中线统一 Feature Store。

本阶段不开发预测模型。

本阶段不开发概率引擎。

本阶段不开发路径分布。

本阶段只负责：

构建高质量特征。

--------------------------------------------------

# 一、设计目标

V1.2 的唯一目标：

把 T 日 14:00 以前的信息，

转换成能够被模型学习的标准特征。

输出：

feature_snapshot_1400.parquet

供未来：

V1.3 Baseline Models

V1.4 Path Distribution

V1.5 Replay

V1.6 Self Learning

统一使用。

--------------------------------------------------

# 二、核心原则

原则1

全部特征必须来自：

T日14:00之前

禁止未来数据。

--------------------------------------------------

原则2

所有特征必须可解释。

禁止：

黑盒特征

随机特征

无法说明来源的特征

--------------------------------------------------

原则3

所有特征必须记录来源。

Feature 必须可追溯。

--------------------------------------------------

原则4

同一个特征只能有一个定义。

禁止重复定义。

--------------------------------------------------

# 六、Feature Manifest

必须建立：

feature_manifest.json

记录：

feature_name

description

source

formula

created_version

--------------------------------------------------

示例：

return_15m

description:

15分钟收益率

formula:

close(t)/close(t-15)-1

--------------------------------------------------
